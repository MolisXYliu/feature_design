# 代码审查技能

当用户要求进行代码审查时，按以下步骤执行：

1. **阅读目标代码** — 使用 read_file 读取需要审查的文件
2. **检查项目**：
   - 代码逻辑是否正确
   - 是否存在潜在的 bug 或边界情况
   - 命名是否清晰、一致
   - 是否有安全隐患（如注入、未验证输入等）
   - 是否遵循项目的编码规范
   - 性能是否有优化空间
3. **输出格式** — 按严重程度分类（严重/警告/建议），每条问题标注文件和行号
