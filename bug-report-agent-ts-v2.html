📋 agent.ts 功能说明与 Bug 分析报告

📌 文件功能概述

src/main/ipc/agent.ts 是 Electron 主进程的 IPC 处理器，负责 AI Agent 的核心交互功能：

  • agent:invoke - 流式调用 AI Agent，处理用户消息并实时返回 token
  • agent:resume - Agent 中断 (HITL) 后恢复执行
  • agent:interrupt - 处理用户对中断请求的批准/拒绝决策
  • agent:cancel - 取消正在运行的 Agent
  • 自动技能进化 - 监控工具调用频率，自动建议生成可复用的 skill

High

1. 事件监听器泄漏 (Event Listener Leak)

位置: agent.ts:425, 975, 1071

// Line 425
window.once("closed", onWindowClosed)

// Line 975
window.once("closed", onWindowClosed)

// Line 1071
window.once("closed", onWindowClosed)

问题: 使用 window.once("closed", ...) 注册了窗口关闭监听器，但这些监听器从未被移除。虽然 once 理论上只触发一次，但在正常流程中窗口不会关闭，导致监听器累积。

修复建议: 在完成流处理后，使用 window.removeListener("closed", onWindowClosed) 显式移除监听器，或使用 AbortSignal 关联的方式管理生命周期。
High

位置: agent.ts:1084

问题: 在 agent:interrupt 处理中，当 decision.type === "approve" 时传入 null 作为输入，这可能导致 Agent 运行时无法正确接收 abort signal，影响资源清理。

修复建议: 确认 LangGraph 的 null 输入是否正确传递 AbortSignal，或显式使用 Command 传递 resume 信号。
