Bug #1: JSON.parse 未捕获异常 高危
📍 位置: 第1070行, 第467行

问题描述

当 thread.metadata 存储了无效的 JSON 字符串时，JSON.parse() 会抛出未捕获的异常，导致整个进程崩溃。

修复方案

// 修复前 (第1070行)
const metadata = thread?.metadata ? JSON.parse(thread.metadata) : {}

// 修复后
let metadata: Record = {}
if (thread?.metadata) {
  try {
    metadata = JSON.parse(thread.metadata)
  } catch (e) {
    console.warn("[Agent] Failed to parse thread metadata:", e)
    metadata = {}
  }
}

Bug #3: interrupt/resume 后缺失技能提案流程 中危
📍 位置: 第1059行 (agent:interrupt), 第959行 (agent:resume)

问题描述

agent:invoke 完成后会调用 autoProposeSKill() 进行技能提案评估，但 agent:interrupt 和 agent:resume 处理完成后均未调用，导致用户中断恢复后的对话无法触发技能提案。

修复方案

在 interrupt 和 resume 的流处理完成后，添加技能提案逻辑：

// 在 interrupt 处理器的 finally 块前添加:
// 检查是否应该触发技能提案
if (shouldEvaluateSkillProposalWindow(threadId)) {
  const context = buildSkillProposalWindowContext(
    snapshotSkillProposalWindow(threadId)
  )
  await autoProposeSKill(threadId, context).catch((e) =>
    console.warn("[Agent] autoProposeSKill failed:", e)
  )
}

同样的逻辑也需要添加到 agent:resume 处理器中。

问题描述

在多线程或高并发场景下，对 activeRuns Map 的读写操作之间可能存在竞态条件。

// 使用锁或原子操作保护
// 方案1: 使用 Map 的同步操作
const abortExistingRun = (threadId: string): AbortController | undefined => {
  const controller = activeRuns.get(threadId)
  if (controller) {
    controller.abort()
    activeRuns.delete(threadId)
  }
  return controller
}

// 方案2: 使用互斥锁 (如果需要更强的保证)
import { Mutex } from 'async-mutex'
const runMutex = new Mutex()

const acquireRunSlot = async (threadId: string) => {
  await runMutex.acquire()
  const existing = activeRuns.get(threadId)
  if (existing) {
    existing.abort()
  }
  const controller = new AbortController()
  activeRuns.set(threadId, controller)
  runMutex.release()
  return controller
}
Bug #5: window.closed 监听器潜在泄漏 低危
📍 位置: 第421行, 第996行, 第1096行

问题描述

使用 window.once("closed", ...) 理论上存在隐患：虽然 once 会自动清理，但如果 window 始终未关闭而 stream 快速完成，可能存在潜在风险。

修复方案

// 方案: 在 finally 块中显式移除监听器 (使用 removeListener)
const onWindowClosed = (): void => {
  console.log("[Agent] Window closed, aborting stream for thread:", threadId)
  abortController.abort()
}
window.once("closed", onWindowClosed)

try {
  // ... stream 处理逻辑
} finally {
  window.removeListener("closed", onWindowClosed)  // 显式移除
  activeRuns.delete(threadId)
}
